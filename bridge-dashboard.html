What Is Bridge?

Bridge is a standalone Python service that connects Telegram to a local Obsidian vault via an intelligent agent. It listens for your messages, talks to a local LLM (LM Studio), saves everything to SQLite for fast context, and asynchronously writes rich markdown notes to your vault with YAML frontmatter, wikilinks, and tags.

The result: a persistent, browsable, queryable second brain that grows automatically as you chat. No manual note‑taking. No copy‑pasting. Just natural conversation → organized knowledge.

⚡ Real‑Time
SQLite WAL ensures zero wait on writes. You get replies instantly.
🧠 Two‑Layer Memory
Hot cache (SQLite) for speed; cold store (Obsidian) for human readability.
🔄 Async Sync
Background queue drains to Obsidian without blocking conversation flow.
🔗 Knowledge Graph
Auto‑extracted facts become atomic notes linked from daily conversations.
📁 Frontmatter & Tags
Every note is Dataview‑compatible. Query, filter, and dashboard in Obsidian.
🌐 REST Fallback
If Obsidian REST plugin is running, vault refreshes live and notes can pop open.

Telegram Commands

/start
Bootstraps first‑run experience or greets returning user with memory preview.
/clear
Wipes session message history. Fresh start without losing long‑term memory.
/status
Shows stats: message count, summary tokens, profile status, knowledge entries, vault path.
/who
Displays agent identity card — name, vibe, soul directive, full IDENTITY.md preview.
/remember <text>
Manually write something to MEMORY.md immediately. Also goes through normal pipeline.
/open <note-path>
If REST plugin configured, opens vault note on your screen (e.g. /open Daily Note/2026-05-02.md).
/sync
Forces immediate sync queue drain instead of waiting for background interval.

Message Pipeline

From Telegram to Obsidian, here's the full path of a single message:

1. Telegram message received → handlers.py:handle_message()
2. User text saved to SQLite (messages table)
3. Context assembled: recent messages + summary + profile
4. Typing indicator task starts (loops every 4s)
5. LLM called via LLMClient.chat() with retry/backoff
6. Bot response saved to SQLite
7. Both user + bot entries formatted & appended to Daily Note
8. _post_process() spawned (non‑blocking):
    • Knowledge extraction → Knowledge/ notes (via LLM)
    • Profile update inference (if needed)
    • Summarization trigger (if message count ≥ 20)
9. Enqueued items written to sync_queue table
10. Background SyncEngine drains queue → formats markdown → writes to vault filesystem
11. If REST configured, vault refresh pinged for live Obsidian update
12. User receives response in Telegram

Module Breakdown

Environment validation; fail‑fast if required vars missing; typed dataclass with defaults.
llm/client.py
Async LM Studio wrapper; exponential backoff retry; both non‑stream and stream modes; typing callback support.
memory/database.py
SQLite with WAL; 6 tables (messages, summaries, knowledge_entries, knowledge_index, user_profile, sync_queue); full async CRUD.
memory/summarizer.py
Triggers at 20+ messages; merges new content into existing summary; extracts knowledge nuggets via LLM (JSON output).
obsidian/vault.py
Direct filesystem R/W; creates directory tree; append/write/delete/read; daily note path resolution; index rebuild.
obsidian/formatter.py
Generates valid Obsidian markdown with proper YAML frontmatter, tags, wikilinks; slugify helper; all note types covered.
obsidian/rest.py
Optional REST plugin client; vault refresh; open note on screen; create/read notes via HTTP.
obsidian/sync.py
Background task; dequeues sync_queue; handles 5 entity types; writes to vault; triggers REST if available.
agent/identity.py
Ensures Agent/IDENTITY.md and Agent/SOUL.md exist; idempotent writes; reads for display.
agent/profile.py
Infers profile updates from recent conversation; upserts to DB + queue; returns new profile text or NO‑CHANGE.
agent/knowledge.py
Extracts 0–3 atomic knowledge nuggets per exchange; stores in DB; queues vault sync.
agent/heartbeat.py
Background scheduler; daily note rollover check; creates placeholders for missing days; extensible for periodic jobs.
bot/handlers.py
BridgeBot class; all 7 commands; message handler with typing loop, context assembly, post‑processing; allowlist middleware.

Quick Start

1. Prerequisites

  • Python 3.11+
  • LM Studio running locally (OpenAI‑compatible API at http://localhost:1234/v1)
  • Telegram Bot Token from @BotFather
  • Your Telegram user ID (add to ALLOWED_USERS)
  • (Optional) Obsidian + REST Plugin for live vault refresh

2. Install & Configure

$ cd bridge
$ python -m venv .venv
$ source .venv/bin/activate # (or .venv\Scripts\activate on Windows)
$ pip install -r requirements.txt
$ cp .env.example .env
Edit .env with your values:
TELEGRAM_BOT_TOKEN=YOUR_TOKEN
ALLOWED_USERS=123456789,987654321
LM_STUDIO_BASE_URL=http://localhost:1234/v1
LM_STUDIO_MODEL=local-model
OBSIDIAN_VAULT_PATH=../BrainVault
AGENT_NAME=Bridge
AGENT_EMOJI=🌉
AGENT_SOUL="You are Bridge, a personal AI assistant..."

3. Run

$ python main.py
[INFO] Initializing Bridge...
[INFO] Bridge initialized successfully
[INFO] Bot is now running

4. Talk to It

Open Telegram, start a chat with your bot, and send any message. Within seconds:

  • Your message + bot reply saved to BrainVault/Daily Note/YYYY-MM-DD.md
  • Knowledge nuggets appear in BrainVault/Knowledge/ (if any extracted)
  • User profile in BrainVault/People/USER.md updates automatically over time
  • Summary in BrainVault/Memory/MEMORY.md grows and gets compressed

First message creates the entire vault structure automatically.

Key Technical Decisions

SQLite as Hot Cache
WAL mode for concurrent reads; no user ever waits on disk I/O to Obsidian; persistent across restarts.
Async Sync Queue
Decouples message processing from vault writes; background task drains every SYNC_INTERVAL; failures retry with backoff.
Typing Indicator Loop
Separate asyncio.Task sends send_chat_action("typing") every 4s until LLM responds — no expired callbacks.
REST Plugin Fallback
Optional live refresh; note opening; graceful degradation if not configured.
LLM Retry Logic
Exponential backoff (2^attempt + jitter); max 3 retries; proper exception handling; no unscoped variables.
Knowledge Graph via Wikilinks
Daily notes link to knowledge, people, memory — true graph brain; Dataview‑queryable via frontmatter.
